{\footnotesize\ttfamily \& D\+B\+\_\+common\+::get\+Assoc (\begin{DoxyParamCaption}\item[{}]{\$query,  }\item[{}]{\$force\+\_\+array = {\ttfamily false},  }\item[{}]{\$params = {\ttfamily array()},  }\item[{}]{\$fetchmode = {\ttfamily DB\+\_\+FETCHMODE\+\_\+DEFAULT},  }\item[{}]{\$group = {\ttfamily false} }\end{DoxyParamCaption})}

Fetches an entire query result and returns it as an associative array using the first column as the key

If the result set contains more than two columns, the value will be an array of the values from column 2-\/n. If the result set contains only two columns, the returned value will be a scalar with the value of the second column (unless forced to an array with the \$force\+\_\+array parameter). A \mbox{\hyperlink{classDB}{DB}} error code is returned on errors. If the result set contains fewer than two columns, a D\+B\+\_\+\+E\+R\+R\+O\+R\+\_\+\+T\+R\+U\+N\+C\+A\+T\+ED error is returned.

For example, if the table \char`\"{}mytable\char`\"{} contains\+:


\begin{DoxyPre}
 ID      TEXT       DATE
-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/-\/
 1       'one'      944679408
 2       'two'      944679408
 3       'three'    944679408
\end{DoxyPre}


Then the call get\+Assoc(\textquotesingle{}S\+E\+L\+E\+CT id,text F\+R\+OM mytable\textquotesingle{}) returns\+: 
\begin{DoxyPre}
  array(
    '1' => 'one',
    '2' => 'two',
    '3' => 'three',
  )
\end{DoxyPre}


...while the call get\+Assoc(\textquotesingle{}S\+E\+L\+E\+CT id,text,date F\+R\+OM mytable\textquotesingle{}) returns\+: 
\begin{DoxyPre}
  array(
    '1' => array('one', '944679408'),
    '2' => array('two', '944679408'),
    '3' => array('three', '944679408')
  )
\end{DoxyPre}


If the more than one row occurs with the same value in the first column, the last row overwrites all previous ones by default. Use the \$group parameter if you don\textquotesingle{}t want to overwrite like this. Example\+:


\begin{DoxyPre}
getAssoc('SELECT category,id,name FROM mytable', false, null,
         DB\_FETCHMODE\_ASSOC, true) returns:\end{DoxyPre}



\begin{DoxyPre}  array(
    '1' => array(array('id' => '4', 'name' => 'number four'),
                 array('id' => '6', 'name' => 'number six')
           ),
    '9' => array(array('id' => '4', 'name' => 'number four'),
                 array('id' => '6', 'name' => 'number six')
           )
  )
\end{DoxyPre}


Keep in mind that database functions in \mbox{\hyperlink{categoryPHP}{P\+HP}} usually return string values for results regardless of the database\textquotesingle{}s internal type.

{\footnotesize\ttfamily D\+B\+\_\+common\+::prepare (\begin{DoxyParamCaption}\item[{}]{\$query }\end{DoxyParamCaption})}

Prepares a query for multiple execution with \mbox{\hyperlink{classDB__common_a668870fe685622c5c394539db4fb5087}{execute()}}

Creates a query that can be run multiple times. Each time it is run, the placeholders, if any, will be replaced by the contents of \mbox{\hyperlink{classDB__common_a668870fe685622c5c394539db4fb5087}{execute()}}\textquotesingle{}s \$data argument.

Three types of placeholders can be used\+:
\begin{DoxyItemize}
\item {\ttfamily ?} scalar value (i.\+e. strings, integers). The system will automatically quote and escape the data.
\item {\ttfamily !} value is inserted \textquotesingle{}as is\textquotesingle{}
\item {\ttfamily \&} requires a file name. The file\textquotesingle{}s contents get inserted into the query (i.\+e. saving binary data in a db)
\end{DoxyItemize}

Example 1. {\ttfamily  \$sth = \$db-\/$>$prepare(\textquotesingle{}I\+N\+S\+E\+RT I\+N\+TO tbl (a, b, c) V\+A\+L\+U\+ES (?, !, \&)\textquotesingle{}); \$data = array( \char`\"{}\+John\textquotesingle{}s text\char`\"{}, \char`\"{}\textquotesingle{}it\textquotesingle{}\textquotesingle{}s good\textquotesingle{}\char`\"{}, \textquotesingle{}filename.\+txt\textquotesingle{} ); \$res = \$db-\/$>$execute(\$sth, \$data); }

Use backslashes to escape placeholder characters if you don\textquotesingle{}t want them to be interpreted as placeholders\+: 
\begin{DoxyPre}
   "UPDATE foo SET col=? WHERE col='over \(\backslash\)\& under'"
\end{DoxyPre}


With some database backends, this is emulated.

\item oci8
