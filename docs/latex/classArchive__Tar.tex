{\footnotesize\ttfamily Archive\+\_\+\+Tar\+::add\+Modify (\begin{DoxyParamCaption}\item[{}]{\$p\+\_\+filelist,  }\item[{}]{\$p\+\_\+add\+\_\+dir,  }\item[{}]{\$p\+\_\+remove\+\_\+dir = {\ttfamily \textquotesingle{}\textquotesingle{}} }\end{DoxyParamCaption})}

This method add the files / directories listed in \$p\+\_\+filelist at the end of the existing archive. If the archive does not yet exists it is created. The \$p\+\_\+filelist parameter can be an array of string, each string representing a filename or a directory name with their path if needed. It can also be a single string with names separated by a single blank. The path indicated in \$p\+\_\+remove\+\_\+dir will be removed from the memorized path of each file / directory listed when this path exists. By default nothing is removed (empty path \textquotesingle{}\textquotesingle{}) The path indicated in \$p\+\_\+add\+\_\+dir will be added at the beginning of the memorized path of each file / directory listed. However it can be set to empty \textquotesingle{}\textquotesingle{}. The adding of a path is done after the removing of path. The path add/remove ability enables the user to prepare an archive for extraction in a different path than the origin files are. If a file/dir is already in the archive it will only be added at the end of the archive. There is no update of the existing archived file/dir. However while extracting the archive, the last file will replace the first one. This results in a none optimization of the archive size. If a file/dir does not exist the file/dir is ignored. However an error text is send to \mbox{\hyperlink{classPEAR}{P\+E\+AR}} error. If a file/dir is not readable the file/dir is ignored. However an error text is send to \mbox{\hyperlink{classPEAR}{P\+E\+AR}} error.

This method creates the archive file and add the files / directories that are listed in \$p\+\_\+filelist. If the file already exists and is writable, it is replaced by the new tar. It is a create and not an add. If the file exists and is read-\/only or is a directory it is not replaced. The method return false and a \mbox{\hyperlink{classPEAR}{P\+E\+AR}} error text. The \$p\+\_\+filelist parameter can be an array of string, each string representing a filename or a directory name with their path if needed. It can also be a single string with names separated by a single blank. The path indicated in \$p\+\_\+remove\+\_\+dir will be removed from the memorized path of each file / directory listed when this path exists. By default nothing is removed (empty path \textquotesingle{}\textquotesingle{}) The path indicated in \$p\+\_\+add\+\_\+dir will be added at the beginning of the memorized path of each file / directory listed. However it can be set to empty \textquotesingle{}\textquotesingle{}. The adding of a path is done after the removing of path. The path add/remove ability enables the user to prepare an archive for extraction in a different path than the origin files are. See also \mbox{\hyperlink{classArchive__Tar_a87c8ae9ed359f1caf7d346649d9155a4}{add\+Modify()}} method for file adding properties.

{\footnotesize\ttfamily Archive\+\_\+\+Tar\+::extract\+Modify (\begin{DoxyParamCaption}\item[{}]{\$p\+\_\+path,  }\item[{}]{\$p\+\_\+remove\+\_\+path }\end{DoxyParamCaption})}

This method extract all the content of the archive in the directory indicated by \$p\+\_\+path. When relevant the memorized path of the files/dir can be modified by removing the \$p\+\_\+remove\+\_\+path path at the beginning of the file/dir path. While extracting a file, if the directory path does not exists it is created. While extracting a file, if the file already exists it is replaced without looking for last modification date. While extracting a file, if the file already exists and is write protected, the extraction is aborted. While extracting a file, if a directory with the same name already exists, the extraction is aborted. While extracting a directory, if a file with the same name already exists, the extraction is aborted. While extracting a file/directory if the destination directory exist and is write protected, or does not exist but can not be created, the extraction is aborted. If after extraction an extracted file does not show the correct stored file size, the extraction is aborted. When the extraction is aborted, a \mbox{\hyperlink{classPEAR}{P\+E\+AR}} error text is set and false is returned. However the result can be a partial extraction that may need to be manually cleaned.
