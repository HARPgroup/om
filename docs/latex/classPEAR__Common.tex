{\footnotesize\ttfamily P\+E\+A\+R\+\_\+\+Common\+::download\+Http (\begin{DoxyParamCaption}\item[{}]{\$url,  }\item[{\&}]{\$ui,  }\item[{}]{\$save\+\_\+dir = {\ttfamily \textquotesingle{}.\textquotesingle{}},  }\item[{}]{\$callback = {\ttfamily null} }\end{DoxyParamCaption})}

Download a file through H\+T\+TP. Considers suggested file name in Content-\/disposition\+: header and can run a callback function for different events. The callback will be called with two parameters\+: the callback type, and parameters. The implemented callback types are\+:

\textquotesingle{}setup\textquotesingle{} called at the very beginning, parameter is a UI object that should be used for all output \textquotesingle{}message\textquotesingle{} the parameter is a string with an informational message \textquotesingle{}saveas\textquotesingle{} may be used to save with a different file name, the parameter is the filename that is about to be used. If a \textquotesingle{}saveas\textquotesingle{} callback returns a non-\/empty string, that file name will be used as the filename instead. Note that \$save\+\_\+dir will not be affected by this, only the basename of the file. \textquotesingle{}start\textquotesingle{} download is starting, parameter is number of bytes that are expected, or -\/1 if unknown \textquotesingle{}bytesread\textquotesingle{} parameter is the number of bytes read so far \textquotesingle{}done\textquotesingle{} download is complete, parameter is the total number of bytes read \textquotesingle{}connfailed\textquotesingle{} if the T\+CP connection fails, this callback is called with array(host,port,errno,errmsg) \textquotesingle{}writefailed\textquotesingle{} if writing to disk fails, this callback is called with array(destfile,errmsg)

If an H\+T\+TP proxy has been configured (http\+\_\+proxy P\+E\+A\+R\+\_\+\+Config setting), the proxy will be used.
